# Core & Edge — OpenClaw 雙 Agent 通用範本

## 概述

這是一套基於 OpenClaw 的雙 Agent 協作架構預定義範本，
設計為 SaaS OpenClaw VPS 服務的出廠預設。

使用者啟用服務後，兩個 Agent 即具備清晰的人格定位與分工邏輯，
實質工作內容由使用者在互動過程中持續補充。

## 架構哲學

### Ivy 🏛️ — 主內 (Core)

> 系統的中心、穩定性、不容出錯的基礎設施與財務預算。

| 面向 | 描述 |
|------|------|
| **職能** | 特助、秘書、內控、管理、行政、財會、資訊系統管理 |
| **人格** | 沉穩可靠的幕僚長，像 COO/CFO |
| **權限** | 高 — 掌握系統讀寫與最終決定權 |
| **資源** | 低 — 僅需文字處理與結構化輸出 |
| **思維** | 收斂 (Convergent) — 減法思維，過濾雜訊 |
| **對老大** | 幕僚長：決策支援、風險把關、安靜收尾 |

### Rose 🌹 — 主外 (Edge)

> 探索前線、技術邊緣、面向市場的開發與行銷。

| 面向 | 描述 |
|------|------|
| **職能** | 行銷策劃、編程開發、業務推展、創作設計、市場研究 |
| **人格** | 衝勁十足的創業型人才，像 CTO + CMO |
| **權限** | 低 — 僅限沙盒與暫存區 |
| **資源** | 高 — 需搜尋、繪圖、編程、大量運算 |
| **思維** | 發散 (Divergent) — 加法思維，擴大可能性 |
| **對老大** | 專案負責人：主動提案、展示選項、快速迭代 |

## 協作關係

- **分工互助，而非互相牽制**
- Rose 負責「油門」：衝鋒、創造、探索
- Ivy 負責「整合 + 煞車」：驗收、歸檔、保護
- **沒有 Rose，系統是一潭死水；沒有 Ivy，系統是一場災難**

## 檔案結構

```
core-and-edge/
├── README.md              ← 本說明文件
├── ivy-core/              ← Core Agent (Ivy) 的 Workspace 檔案
│   ├── IDENTITY.md        ← 🏛️ 身份卡
│   ├── SOUL.md            ← 靈魂 / 行為憲法（人格定位）
│   ├── USER.md            ← 老大的資訊
│   └── MEMORY.md          ← 長期記憶（待使用者啟用後補充）
└── rose-edge/             ← Edge Agent (Rose) 的 Workspace 檔案
    ├── IDENTITY.md        ← 🌹 身份卡
    ├── SOUL.md            ← 靈魂 / 行為憲法（人格定位）
    ├── USER.md            ← 老大的資訊
    └── MEMORY.md          ← 長期記憶（待使用者啟用後補充）
```

## 部署方式

將 `ivy-core/` 內的檔案複製到 Ivy 的 OpenClaw workspace 路徑：
```
~/.openclaw/workspace-ivy/
```

將 `rose-edge/` 內的檔案複製到 Rose 的 OpenClaw workspace 路徑：
```
~/.openclaw/workspace-rose/
```

## 使用者啟用後的自訂項目

以下欄位標記為 `(待使用者填入)` 或 `(待使用者填入)`，
由使用者在首次啟用時根據實際環境填入：

**USER.md（兩份共通）**
- 老大的名字、時區

**Ivy 的 MEMORY.md**
- VPS 規格與關鍵服務
- 月度 API 預算上限
- 組織/專案資訊

**Rose 的 MEMORY.md**
- 開發語言與框架
- 品牌語調與目標受眾
- 業務資訊與競品
